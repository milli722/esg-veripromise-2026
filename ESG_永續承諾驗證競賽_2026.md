## 一、 競賽簡介

隨著全球 **永續發展目標 (SDGs)** 與 **淨零碳排** 政策的推動，企業揭露 ESG 承諾已成為衡量其責信與治理能力的重要指標。然而，許多永續報告書內容模糊且缺乏實證，引發「**漂綠 (Greenwashing)**」疑慮。

為推動企業永續資訊揭露的透明化與可信度，本競賽提供 4,000 筆源自真實產業情境的繁體中文 ESG 資料集 **「VeriPromiseESG4K 標註語料庫」**。參賽者將運用自然語言處理 (NLP) 與大型語言模型 (LLM) 等 AI 技術，開發能自動識別、分析和驗證企業永續承諾的模型。

> **核心目標**：發展具備邏輯識別與可檢驗性的 NLP 技術，提升繁體中文永續資訊的品質，以 AI 具體回應真實世界的永續課題。

## 三、 核心任務與評估標準

本競賽參考 ML-Promise 國際研究成果，採用多任務綜合評分機制。針對企業承諾驗證的四個核心任務進行差異化權重設計：

| 任務編號 | 任務名稱 | 任務說明 | 評分指標 | 評分權重 |
| :--- | :--- | :--- | :--- | :---: |
| **子任務一** | **承諾語句識別** <br>(Commitment Classification) | 判斷給定的文本是否表達企業對未來行動的明確承諾。 | F1-Score | **20%** |
| **子任務二** | **預期驗證時機推論** <br>(Timeline Classification) | 根據語句推論承諾的預期完成時間（四分類預測）。 | Macro-F1 Score | **15%** |
| **子任務三** | **支持證據連結** <br>(Evidence Identification) | 判斷已識別的承諾語句是否附有具體執行計畫或支持佐證。 | F1-Score | **30%** |
| **子任務四** | **證據清晰度評估** <br>(Clarity Classification) | 評估「承諾-證據」對的語意清晰度，是否無模糊字眼（三分類預測）。| Macro-F1 Score | **35%** |

## 四、 參賽對象與報名

### 1. 參賽對象身分
- **學生組**：報名日起至競賽測試集答案上傳截止，具備中華民國學籍、年滿 18 歲之各級校院在學生（未滿 18 歲需法定監護人同意）。敘獎時須檢附學生證影本或在學證明正本。
- **社會人士組**：隊伍內若有 **任一名** 非符合上述學生組身分者（含國內外社會人士或不受中華民國學籍限制之學生），即全隊列為社會人士組。

### 2. 報名方式
1. 於 **「AI CUP 報名系統」** 登入並依指引完成組隊登錄（隊名由系統隨機給定）。
2. 報名後 1-2 個工作日內將收到 **「AIdea 平臺」** 帳號開通通知，依信件指示完成開通。
3. 若配合課程參賽，請填寫 **「課程代碼」**。
4. 報名前需填寫「前測問卷」，賽後需填寫「後測問卷」。

## 七、 資料集與欄位說明

釋出之訓練集、驗證集及測試集存取於 `data.zip`。每筆資料為單一標註樣本（JSON格式），欄位定義如下：

```json
{
  "data": "節錄自各企業 2024 年 ESG 永續報告書之原始段落文本",
  "URL": "企業永續報告書原始來源超連結",
  "page_number": "對應來源中的頁碼",
  "ESG_type": "E(環境)/S(社會)/G(公司治理)，可多選",
  "promise_status": "Yes(包含明確承諾) / No(不包含承諾)",
  "promise_string": "摘錄之承諾語句 (若無則為空字串)",
  "verification_timeline": "already / within_2_years / between_2_and_5_years / more_than_5_years / N/A",
  "evidence_status": "Yes(包含明確證據) / No(不包含證據) / N/A",
  "evidence_string": "摘錄之證據語句 (若無則為空字串)",
  "evidence_quality": "Clear(清晰) / Not Clear(不夠清晰) / Misleading(誤導性) / N/A"
}
```

## 八、 競賽規則與注意事項

> [注意] **違規處置**：如有以下情事，主辦方得**逕行取消**參賽或領獎資格，若已發放獎勵亦得於賽後追討：
> 1. 抄襲、作弊或詐欺行為。
> 2. 侵害他人智慧財產權。
> 3. 對排行榜系統進行攻擊，或影響其他隊伍導致不公平。
> 4. 違反報名系統或平臺使用者條款。

- **上傳限制**：測試集預測結果**每日提交上限為 3 次**，Leaderboard 以最後一次結果呈現。
- **禁止人工修正**：嚴禁對測試資料集或辨識結果進行**任何形式的人工標註或修正**，預測結果須由程式自動生成。
- **程式獨立性**：禁止私下共享程式與特徵值，報告應獨立撰寫。（官方討論區之公開內容不在此限）
- **逾期無效**：報告、程式碼與預測結果皆**不接受遲交或補交**，未於期限內成功繳交者視同棄賽。
- **參賽授權**：競賽期間隊員資料與智慧財產權屬參賽隊伍所有，但須無償授權主辦單位進行宣傳、紀錄及成果發表。
